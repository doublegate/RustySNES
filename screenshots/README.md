# screenshots/

Boot captures of the whole locally-staged ROM corpus. Generated by
`scripts/screenshots/generate.sh` (which runs the `commercial_screenshots` harness + ImageMagick).

Every ROM under `tests/roms/` is booted on the full core and captured **twice**:

- `<name>_title.png` — the best-content frame in an early window (no controller input): the game's
  title / logo / intro screen.
- `<name>_gameplay.png` — the best-content frame in a late window, after a rotating Start/A/dir
  button sequence mashes past the title, menus, and intro into gameplay / the attract demo.

"Best-content frame in a window" (most non-black pixels) is used instead of a fixed frame so the
capture dodges the black scene-transition frames a fixed point would randomly land on. Frame
windows are overridable via `RUSTYSNES_TITLE_LO/HI` and `RUSTYSNES_PLAY_LO/HI`.

Layout mirrors the ROM tree under `tests/roms/`:

- `external/commercial/<map>/<chip>/` — commercial dumps (LoRom/HiRom × None/DSP-1/GSU/SA-1/…).
- `external/krom/…`, `external/blargg-spc/`, `undisbeliever/`, `gilyon/cputest/` — test-ROM suites
  (patterns/diagnostics; both frames may look alike).
- `montage_title.png` / `montage_gameplay.png` — full-corpus showcase grids, auto-generated
  alongside every other capture here.

## README images (hand-curated, not auto-generated)

`montage.png` and `showcase.png` at this top level are a **hand-picked** subset of the corpus
above, composited with ImageMagick `montage` and referenced directly by the root
[`README.md`](../README.md) — mirroring RustyNES's own `screenshots/montage.png` /
`screenshots/showcase.png` convention. Unlike `montage_title.png`/`montage_gameplay.png`, these
are **not** regenerated by `scripts/screenshots/generate.sh`; re-curate them by hand (with
`magick montage …`) if the selection should change, e.g. to reflect a newly-added coprocessor or
a better capture for an existing title.

Notes: a heuristic sweep can't perfectly hit every game's exact title (some land on a publisher
logo or intro), and a few ROMs render nothing here (e.g. Stunt Race FX — a known Super FX gap), so
their captures are blank.

Only these PNGs are committed (derived emulator output); the ROMs + coprocessor firmware that
produced them stay gitignored under `tests/roms/external/` (ADR 0003 — never commit ROM bytes).
Regenerate with `scripts/screenshots/generate.sh`.
